Servidor

Em informática, um servidor é um software ou um computador com um sistema de computação centralizada que fornece serviços a uma rede de computadores, chamada de cliente.
Esses serviços podem ser de naturezas distintas, como, por exemplo, arquivos e correio eletrônico.
Esta arquitetura é chamada de modelo cliente-servidor, é utilizada em redes de médio e grande porte (com muitas máquinas) e em redes onde a questão da segurança desempenha um papel de grande importância. O termo “servidor” é amplamente aplicado a computadores completos, embora um servidor possa equivaler a um software, a partes de um sistema computacional ou até mesmo a uma máquina que não seja necessariamente um computador.
Os servidores podem fornecer várias funcionalidades, muitas vezes chamadas de “serviços”, tais como a partilha de dados ou de recursos do sistema entre vários clientes, ou a realização de cálculos de alto desempenho para um cliente. Um único servidor pode atender a vários clientes, e um único cliente pode usar vários servidores. Um processo cliente pode ser executado no mesmo dispositivo ou pode se conectar, através de uma rede, a um servidor em um dispositivo diferente.

História

O uso do termo “servidor” na informática começou com a Teoria das Filas, que remonta a meados do século XX, sendo notavelmente utilizada por Kendall em 1953 (junto com “serviço”), no trabalho que introduziu a notação de Kendall. Em trabalhos anteriores, como o de Erlang em 1909, termos mais concretos, como “operadores [de telefone]”, são usados.
Em informática, o termo “servidores de dados” é encontrado, pelo menos, na RFC 5 de 1969, um dos primeiros documentos a descrever a ARPANET (a predecessora da Internet), e é contrastado com “usuário”, distinguindo dois tipos de hospedeiros: “server-host” e “user-host”. O uso de “servir” também remonta a documentos antigos, como a RFC 4, que contrasta “servidor-host” com “utilizador-host”.
A história dos servidores está, obviamente, ligada às redes de computadores. As redes permitiam a comunicação entre diversos computadores e, com o crescimento destas, surgiu a ideia de dedicar alguns computadores para prestar serviços à rede, enquanto outros utilizariam esses serviços. Os servidores ficariam responsáveis pela primeira função.
Com o advento das redes, cresceu a necessidade de ter servidores e minicomputadores, o que acabou contribuindo para a diminuição do uso dos mainframes.
O crescimento das empresas de redes e o aumento do uso da Internet entre profissionais e usuários comuns foram o grande impulso para o desenvolvimento e aperfeiçoamento das tecnologias de servidores.

Tipos de servidores

Existem diversos tipos de servidores, entre os mais conhecidos estão:

  • Servidor de Fax: Responsável pela transmissão e recepção automatizada de faxes pela Internet, permitindo também o envio, recebimento e distribuição de faxes para todas as estações conectadas.
  • Servidor de Arquivos: Armazena arquivos de diversos usuários.
  • Servidor Web: Responsável pelo armazenamento de páginas de um site específico, acessadas pelos clientes por meio de navegadores.
  • Servidor de E-mail: Gerencia o armazenamento, envio e recebimento de mensagens de correio eletrônico.
  • Servidor de Impressão: Controla os pedidos de impressão de arquivos dos clientes.
  • Servidor de Banco de Dados: Possui e manipula informações contidas em um banco de dados.
  • Servidor DNS: Responsável pela conversão de endereços de sites em endereços IP e vice-versa.
  • Servidor Proxy: Atua como um intermediário, armazenando páginas da internet recentemente visitadas para acelerar o carregamento em acessos subsequentes.
  • Servidor de Imagens: Especializado em armazenar imagens digitais.
  • Servidor FTP: Permite o acesso de usuários a um disco rígido ou servidor, armazenando arquivos para acesso via internet.
  • Servidor Webmail: Permite a criação de e-mails na web.
  • Servidor de Virtualização: Facilita a criação de máquinas virtuais, conhecidas como VPS, para hospedagem de aplicações, otimizando o uso do hardware e a eficiência energética.
  • Servidor de Sistema Operacional: Compartilha o sistema operacional de uma máquina com outras na mesma rede, dispensando a necessidade de um sistema operacional instalado ou um HD próprio em cada uma.
  • Servidor de Aplicações (Application Server): Fornece um ambiente para instalação e execução de aplicações, centralizando o processo e eliminando a necessidade de instalação em computadores clientes. Também conhecido como middleware.

Clientes e servidores se comunicam através de protocolos de rede.
Um computador pode fornecer mais de um serviço simultaneamente. Por exemplo, pode atuar como servidor web e de banco de dados ao mesmo tempo, ou como servidor de arquivos, de e-mail e proxy. Quando um computador desempenha um único tipo de serviço, é chamado de servidor dedicado, o que pode resultar em respostas mais rápidas às requisições dos clientes.
Em redes heterogêneas, um cliente pode ser um servidor e vice-versa. Por exemplo, em uma rede com um servidor de impressão e um de arquivos, se você estiver no servidor de arquivos e precisar imprimir um documento, ao enviar o comando de impressão, o servidor de arquivos atuará como cliente do servidor de impressão.

Hardware e Software de Servidores

Hardware

Servidores dedicados, que têm alta demanda de dados por parte dos clientes e operam em aplicações críticas, utilizam hardware específico para servidores. Já aqueles que não têm essas funções podem usar hardware de um computador comum.
Os servidores geralmente se baseiam em entradas e saídas de informações, como gravações e exclusões de arquivos, o que requer interfaces de entrada e saída e discos rígidos de alto desempenho e confiabilidade. O padrão SCSI é comum para discos rígidos, permitindo a conexão de vários periféricos em arranjos RAID.
Devido ao grande volume de entradas e saídas de informações, os servidores precisam de processadores de alta velocidade e, em alguns casos, são multiprocessados, ou seja, têm mais de um processador. Eles também dispõem de uma grande quantidade de memória RAM, geralmente usada para cache de dados.
Por operarem por longos períodos, frequentemente de forma ininterrupta, alguns servidores são conectados a geradores elétricos. Outros usam sistemas de alimentação, como o UPS, que mantêm o servidor funcionando em caso de queda de tensão.
Além disso, pela necessidade de operar por longos intervalos e devido à presença de um ou mais processadores de alta velocidade, os servidores requerem sistemas eficientes de dissipação de calor, o que implica em coolers mais caros, porém mais eficientes e confiáveis.
Existem outros hardwares específicos para servidores, como placas do tipo hot swapping, que permitem a troca de componentes com o computador ligado, essencial para a continuidade da operação da rede. Há debates sobre a viabilidade de usar um micro comum, como um PC, como servidor, e se é necessário adquirir um equipamento mais robusto. A decisão depende do uso do equipamento e da criticidade do serviço executado pelo servidor. Em ambientes não críticos, um PC comum pode atuar como servidor. O tamanho da rede é irrelevante; por exemplo, uma empresa com três instrutores online pode ter três computadores, sendo um deles o servidor de acesso à Internet. Se esse servidor falhar, a operação da empresa é interrompida.
Antecipando essa necessidade, fabricantes de componentes de computadores desenvolvem placas mais robustas, com engenharia de ventilação mais elaborada, redundância de itens e maior capacidade de expansão, para assegurar a disponibilidade e confiabilidade do serviço.
Geralmente, o desenvolvimento de servidores é focado em grandes fabricantes do mercado, que contam com equipes especializadas e laboratórios dedicados a essa finalidade.

Sistemas Operacionais para Redes Cliente-Servidor

Para o funcionamento de uma rede cliente-servidor, é essencial que o servidor tenha instalado um sistema operacional compatível com esse tipo de rede. Os sistemas operacionais adequados para redes cliente-servidor incluem:

  • Unix
  • Linux
  • Solaris
  • FreeBSD
  • Novell Netware
  • Windows NT, Windows 2000, Windows 2003, Windows XP, Windows Vista, Windows Server 2008, Windows Server 2012, Windows 7, Windows 8 e Windows 10.

Os sistemas operacionais Windows 95, Windows 98 e Windows ME são projetados apenas para redes do tipo ponto-a-pontos, enquanto o sistema operacional DOS não oferece suporte a redes. Nos servidores, os sistemas Unix e os baseados nele, como Linux e Solaris, são frequentemente utilizados para aplicações como firewalls e servidores web. Já os sistemas Windows são preferidos para gerenciamento de usuários e serviços devido à facilidade de operação e manutenção, como o Active Directory (AD).

Servidores na Internet

A Internet, a maior rede de computadores do mundo, opera sob o modelo cliente-servidor. Inúmeros servidores interconectados processam informações de maneira simultânea globalmente. Os servidores de internet disponibilizam diversos serviços, tais como:

  • Páginas web
  • Correio eletrônico
  • Transferência de arquivos
  • Acesso remoto
  • Mensagens instantâneas, entre outros

É notável que qualquer operação realizada por um usuário na internet aciona múltiplos servidores ao redor do planeta, trabalhando em conjunto para executar a tarefa solicitada.
